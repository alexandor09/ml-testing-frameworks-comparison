# DEMO (≤5 минут): структура GitHub + запуск в Google Colab

Цель демонстрации: **показать работоспособность пайплайна**, полный прогон **4 фреймворков**, формирование **дашборда** и построение **mini-table** (E_test vs EF_test → итоговая таблица).

## 1) GitHub: структура и код (≈2 минуты)

### 1.1 Открыть корень репозитория (вкладка Code)
Откройте список файлов/папок и покажите, что проект состоит из:
- `data/` — входные датасеты
- `generate_scenarios.py` — генерация сценариев (синтетика)
- `main.py` — основной CLI пайплайн
- `src/frameworks/` — адаптеры фреймворков
- `scripts/` — mini-table (E_test/EF_test/final)
- `reports/` — артефакты запусков и таблицы

Фраза:
«Проект реализует фиксированный протокол: данные → временной split → Prophet → проверки 4 фреймворками → отчёты и сводные таблицы».

### 1.2 Открыть `generate_scenarios.py`
Что показать:
- что сценарии сохраняются в `data/`
- упоминание сценариев `ideal`, `pass`, `dr`

Фраза:
«Сценарии синтетических данных задаются здесь. `ideal` — контрольный набор, `pass` — пропуски и дубликаты, `dr` — дрейф и выбросы. Важно: изменения для `dr` делаются в тестовом хвосте, потому что split фиксированный 80/20 по времени».

### 1.3 Открыть `data/`
Что показать:
- `ideal.csv`, `pass.csv`, `dr.csv` (можно кратко отметить, что есть и `.json`)

Фраза:
«Это входные данные для эксперимента. Формат колонок: `ds`, `y`, `price`, `promotion`».

### 1.4 Открыть `main.py`
Что показать:
- что используется **time split 80/20** (последние 20% — test)
- что запускаются адаптеры и создаётся папка `reports/<run>/<timestamp>/...`

Фраза:
«`main.py` — CLI пайплайн: сортировка по `ds`, split 80/20, обучение Prophet, прогноз `yhat`, затем запуск адаптеров фреймворков и сохранение артефактов в `reports`».

### 1.5 Открыть `src/frameworks/`
Что показать:
- `gx_adapter.py`, `evidently_adapter.py`, `alibi_adapter.py`, `nannyml_adapter.py`

Фраза:
«Для каждого средства есть адаптер. Результаты унифицированы: время, пик памяти, количество детекций и значения проверок, а также список артефактов».

### 1.6 Открыть `scripts/` (что добавлено нового)
Что показать:
- `count_E_test.py`, `count_EF_test.py`, `build_mini_table.py`
- (опционально) `reports/mini_table_provenance.md`

Фраза:
«Это добавленный функционал mini-table. `E_test` — сколько дефектов внедрено на test split; `EF_test` — что нашли фреймворки по их артефактам; итоговая таблица даёт EF_total/EF_true/FP/Recall/Precision/FalseAlarm».

## 2) Google Colab: запуск полного эксперимента (≈3 минуты)

### 2.1 Открыть Colab-ноутбук
Откройте `colab_demo.ipynb` в Colab (через ссылку из README или вручную).

Действие:
- В Colab: **Runtime → Run all** (или выполнить ячейки сверху вниз).

Фраза:
«Сейчас запускается воспроизводимый эксперимент: полный прогон 4 фреймворков по сценариям `ideal/pass/dr`, затем сборка mini-table».

### 2.2 Что именно показать после выполнения ячейки “полный эксперимент”
В ноутбуке будет:
- таблицы сравнения из `comparison_summary.json` (time/memory/issues/coverage)
- сборка mini-table и печать путей к файлам:
  - `reports/mini_table_E_test.json`
  - `reports/mini_table_EF_test.csv`
  - `reports/mini_table_final.csv`

Фраза:
«Это сравнение по ресурсам и количеству найденных проблем, а также итоговая mini-table с FP/Recall/Precision. Все значения формируются автоматически и сохраняются в `reports/`».

### 2.3 Дашборд (HTML)
Действие:
- если Colab предложил скачать `dashboard.html` — скачайте и откройте локально в браузере.

Фраза:
«В `dashboard.html` собраны графики сравнения фреймворков и ссылки на артефакты. Это итоговый отчёт для визуальной проверки результата эксперимента».
